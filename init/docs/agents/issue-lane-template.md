# Agent Lane Issue Template

Use this structure when opening a new implementation lane.

## Summary

One paragraph describing the lane outcome and why it matters now.

## Scope

- exact behaviors to add or change
- files or subsystems likely involved
- what this lane intentionally does not include

## Acceptance criteria

- observable end state
- exact tests or smoke checks expected
- any docs that must be updated

## Out of scope

- adjacent follow-ups
- future polish
- related lanes that should stay separate

## Suggested branch and worktree

- branch: `codex/issue-<n>-<slug>`
- worktree: `.worktrees/<n>-<slug>`
