# Specs

Feature specifications and design notes for work that is planned or in progress.

## How To Use

- One file per feature or significant change (e.g. `auth-oauth.md`).
- A spec is a living document while the work is active: keep it in version
  control so humans and agents share one source of truth, and update it when
  scope or decisions change.
- The `spec-driven-development` and `planning-and-task-breakdown` skills write
  here. Link the active spec from `../../TASKS.md` so a new session can find it.
- Once a feature ships, either keep the spec as reference or delete it — nothing
  depends on it remaining.
