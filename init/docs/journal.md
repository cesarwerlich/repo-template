# Journal

Chronological notes for direction changes, discoveries, and discussion outcomes
that should be written down but do not yet belong in `memory.md`, `DECISIONS.md`,
an ADR, or a runbook.

## How To Use

- Record dated, high-signal notes — not step-by-step execution logs. Use
  `git log` for what changed line by line.
- Promote stable truths and repeated decisions into `memory.md`; promote settled
  choices into `../DECISIONS.md` or `adr/`.
- Link to deeper docs when a note grows beyond a quick entry.
- When this file gets long, move older entries into `history/`.

## Entries

```text
YYYY-MM-DD - Topic
- Context:
- What changed:
- Decision:
- Follow-up:
```
