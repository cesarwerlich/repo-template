# Agent Subsystem

Optional guidance for AI coding agents.

## How To Use

1. Read the project root `CONTEXT.md`, `ROADMAP.md`, and `MEMORY.md`.
2. Read `TESTING.md` and `EVALS.md` when the task touches verification, benchmarks, or repeatable quality checks.
3. Use `docs/agents/` for tool-neutral personas and playbooks.
4. Pick a relevant skill from `skills/` only after understanding the task.
5. Use personas in `agents/` for focused review perspectives when the repo actually needs them.
6. Use `references/` for checklists when a skill asks for deeper support.

Skills and personas are helpers. Local project instructions in `AGENTS.md` always take precedence.
