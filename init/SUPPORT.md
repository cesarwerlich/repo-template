# Support

## Start Here

Before asking for help, check:

- `README.md`
- `CONTEXT.md`
- `docs/roadmap.md`
- `docs/memory.md`
- `docs/testing.md`
- `docs/evals.md`

## How To Ask For Help

Use issues for normal questions, bugs, and feature requests unless the repository owner documents another path.

For security-sensitive concerns, use `SECURITY.md` instead of opening a public issue.

## Good Support Requests

Include:

- what you were trying to do
- the command or workflow you ran
- the expected result
- the actual result
- relevant logs, screenshots, or error output
